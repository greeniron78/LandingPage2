# landing_page2 Docs

## 읽는 순서

1. `../AGENTS.md`
2. `mvp-spec.md`
3. `agent-rules.md`
4. `design/storyboard.md`
5. 현재 작업 TASK 파일

## 문서 역할

- `mvp-spec.md`: 제품 목표, 기술 구조, MVP 범위
- `agent-rules.md`: Codex 작업 및 토큰 절약 규칙
- `design/storyboard.md`: 페이지 흐름과 스크롤 연출 기준
- `development/task-template.md`: 기능 TASK 작성 양식
- `tasks/`: 기능별 작업 명세

## 운영 원칙

- 새 기능은 TASK 파일로 작성한다.
- 공통 규칙은 TASK에 반복하지 않고 기준 문서에만 작성한다.
- 디자인 또는 기술 결정이 확정될 때만 관련 문서를 추가한다.
- 문서가 충돌하면 `mvp-spec.md`를 우선한다.
