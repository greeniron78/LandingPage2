# Storyboard — landing_page2

## 페이지 목적

사용자가 스크롤을 통해 핵심 메시지와 이야기를 이해하고, 마지막 CTA 행동으로 이어지는 모바일 우선 스토리텔링 랜딩 페이지를 만든다.

## 기본 스크롤 구조

1. Hero
   - 첫 메시지와 스크롤 유도
   - 배경 이미지 또는 비주얼 사용 가능
   - 주요 CTA 선택 가능

2. Video
   - 브랜드 또는 캠페인 메시지를 전달하는 YouTube 영상
   - 영상이 없으면 섹션 전체를 숨길 수 있어야 함

3. Concept
   - 서비스 또는 캠페인의 핵심 관점 설명
   - 짧은 제목, 설명, 강조 문장으로 구성

4. Sticky Story
   - 배경 레이어는 sticky 상태로 유지
   - 카드와 텍스트 콘텐츠만 세로 스크롤
   - 카드마다 하나의 이야기 또는 핵심 메시지 제공
   - 모바일에서 카드가 읽기 쉽고 터치하기 쉬워야 함

5. Solution
   - 제공 가치, 해결 방식, 참여 방법을 설명

6. CTA
   - 카카오톡, 전화, 길찾기, 외부 링크 등 행동 유도
   - 페이지 하단과 플로팅 CTA 모두 지원

7. Footer
   - 기본 안내, 연락처, 정책 링크

## Sticky Story 구현 원칙

- sticky 배경과 스크롤 콘텐츠를 분리한다.
- 배경은 시각적 맥락을 제공하고, 텍스트 가독성을 방해하지 않는다.
- 카드의 수와 내용은 데이터 파일에서 관리한다.
- 초기 MVP는 복잡한 스크롤 애니메이션 없이 CSS sticky와 자연스러운 스크롤만 사용한다.
- 모션과 카드 전환 효과는 별도 TASK에서 추가한다.

## 반응형 기준

- 모바일을 우선으로 설계한다.
- 데스크톱은 콘텐츠 폭을 제한하고 여백을 확장한다.
- 고정 CTA는 모바일 화면의 안전 영역과 콘텐츠 하단 여백을 고려한다.

## 디자인 결정 기록

- 확정된 색상, 폰트, 이미지 방향, 카드 스타일은 필요 시 `design-system.md`에 추가한다.
- 확정된 애니메이션 규칙은 필요 시 `motion-spec.md`에 추가한다.
